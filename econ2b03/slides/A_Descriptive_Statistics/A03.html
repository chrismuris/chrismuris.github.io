Part I: One Categorical Variable

Data presentation

  • In SZ, we will see at least two ways of presenting data:
    • A data list is an explicit listing of all the individual measurements made on a sample.
    • A data frequency table is a table listing each distinct value \(x\) and its frequency.
  • The frequency of a value \(x\) is the number of times it appears in the sample data.
  • The following slide combines text, code, and output
  • It is here to help you get used to reading R code
  • It is generated from quarto, just like your assignments:
    • You can recognize code from the font and syntax highlighting
    • Output is preceded by ##
  • You can copy-paste-modify for your assignments

Example

  • When you are writing a report, you may want a nicer version of a frequency table.
  • You can use the janitor and gt packages.
library(janitor)
library(gt)
penguins |> tabyl(sex) |> gt()
sex n percent
female 165 0.4954955
male 168 0.5045045
  • We distinguish between absolute and relative frequencies
  • The absolute frequency, or frequency, is the number of measurements in the data set that are equal to a given value
  • The absolute frequency distribution is a tabular summary of a data set that shows the absolute frequency for each value in the sample data
  • The relative frequency, or proportion, is the absolute frequency divided by the sample size \(n\)
  • The relative frequency distribution is a tabular summary of a data set that shows the relative frequency for each value in the sample data
  • Our janitor/gt frequency table gave us both:
penguins |> filter(!is.na(sex)) |> tabyl(sex) |> gt()
sex n percent
female 165 0.4954955
male 168 0.5045045
  • In R, there is usually multiple ways to compute something.
  • For example, to obtain relative frequencies using table instead of tabyl:
nrow(penguins)
[1] 333
table(penguins$species)

   Adelie Chinstrap    Gentoo 
      146        68       119 
table(penguins$species) / nrow(penguins)

0.4384384 0.2042042 0.3573574

Part II: Two Categorical Variables

Two categorical variables

  • We will use the loan50 data from the openintro package that comes with IMS.

data on loans from Lending Club that you’ve previously seen in Chapter 1. The loan50 dataset from Chapter 1 represents a sample from a larger loan dataset called loans. This larger dataset contains information on 10,000 loans made through Lending Club. We will examine the relationship between homeownership, which for the loans data can take a value of rent, mortgage (owns but has a mortgage), or own, and app_type, which indicates whether the loan application was made with a partner or whether it was an individual application.

library(openintro)
data("loans_full_schema")
loans <- loans_full_schema |> 
  select(state, annual_income, verified_income, homeownership, application_type) |> drop_na() |> 
  filter(application_type %in% c("individual","joint")) |>
  filter(homeownership %in% c("MORTGAGE","OWN","RENT")) |>
  mutate(
    application_type = fct_drop(application_type),
    homeownership = fct_drop(homeownership),
    homeownership = fct_recode(homeownership,
      mortgage = "MORTGAGE",
      own = "OWN",
      rent = "RENT"
    )
  )
#?loan50
  • We will focus on the following two variables:
    • application_type: is the loan application made by one person, or with a partner?
    • homeownership: does the applicant (team) own their home, own with a mortgage, or do they rent?
  • We are interested in the association between these two variables
  • We have access to a data set of 10000 loans made by The Lending Club
  • A table that summarizes data for two categorical variables is called a contingency table
  • Each value is the number of times a particular combination of variable outcomes occurred
loans |> 
  tabyl(homeownership, application_type) |> 
  gt()
homeownership individual joint
mortgage 3839 950
own 1170 183
rent 3496 362
  • To add totals, ask ?tabyl or Google to find:
loans |> 
  tabyl(homeownership, application_type) |>
  adorn_totals(where = c("row","col"))
 homeownership individual joint Total
      mortgage       3839   950  4789
           own       1170   183  1353
          rent       3496   362  3858
         Total       8505  1495 10000
  • pipe into gt for nice output:
loans |> 
  tabyl(homeownership, application_type) |>
  adorn_totals(where = c("row","col")) |>
  gt()

  • Even with the totals, it is hard to uncover any association from this table.
  • There are two things we can do to help:
    1. visualize
    2. use row and column proportions
  • To visualize the association between two variables, we can use bar plots:
    • stacked
    • dodged
    • standardized (or filled)
  • What do we learn from this stacked bar plot?
loans |> ggplot() + 
  geom_bar(
    aes(
      x=homeownership, 
      fill=application_type)
  )
  • Findings:
    • most loans are mortgage loans, relatively few people own
    • across all types, most applications are individual
    • hard to see if there is an association between the two variables
  • Do you learn anything else from this dodged bar plot?
loans |> ggplot() + 
  geom_bar(
    aes(x=homeownership, 
        fill=application_type), 
        position = position_dodge()
  )

Not really. It is easier to see the raw counts. It is easy to see that, across all types of homeownership, individual is more commont han joint. It seems like the ratio of individual to joint is much higher for rent.

  • How about this standardized or filled bar plot?
loans |> ggplot() + 
  geom_bar(
    aes(x=homeownership, 
        fill=application_type), 
        position = position_fill()
  )

This one is very handy for seeing that, among applications for renting, joint applications are less common. Does it make sense? I find the own type a bit surprising.

  • The filled bar plot gives us an idea of the distribution of one variable (application_type) for a given value of homeownership
  • This is a useful way to think about the association between the two variables
  • The previous plot suggests that, for mortgages, joint application types are more likely
  • To achieve the same in a contingency table, we can use row and column proportions
loans |> tabyl(homeownership, application_type) |>
  adorn_percentages(denominator = "row") |>
  adorn_totals(where = c("col")) |>
  gt()

mortgage 0.8016287 0.1983713 1
own 0.8647450 0.1352550 1
rent 0.9061690 0.0938310 1

loans |> tabyl(homeownership, application_type) |>

Exercises

Exercise: views on immigration

Nine-hundred and ten (910) randomly sampled registered voters from Tampa, FL were asked about views on immigration policy.

  • Question (5 min)
  • Answers

Use the contingency table below to answer the questions.

Response Conservative Liberal Moderate Total
Apply for citizenship 57 101 120 278
Guest worker 121 28 113 262
Leave the country 179 45 126 350
Not sure 15 1 4 20
Total 372 175 363 910
  1. What percent of these voters identify as conservatives?
  2. What percent are in favor of the citizenship option?
  3. What percent are conservative and in favor of the citizenship option?
  4. Among conservatives, moderates, and liberals, what percent support the citizenship option?
  5. Do political ideology and views on immigration appear to be associated?
  1. Conservatives: \(372/910 \approx 0.409\).
  2. Citizenship option: \(278/910 \approx 0.305\).
  3. Conservative and citizenship: \(57/910 \approx 0.063\).
  4. Conditional support:
    • Conservatives: \(57/372 \approx 0.153\)
    • Moderates: \(101/175 \approx 0.577\)
    • Liberals: \(120/363 \approx 0.331\)
  5. Yes. Support varies substantially by political ideology, indicating association.
